# CLAUDE.md

## コア原則

- **事実から始める**: 推測で動かない。まずログ・エラー・スタックトレースを確認する。
- **根本から正しく解決する**: 応急処置やハック的な修正に逃げない。根本原因を特定し、よりよい方法がないか考えたうえで解決する。
- **シンプル・最小限**: 変更は可能な限り単純に。必要な箇所だけ変更し、過剰な設計・修正をしない。
- **立ち止まって見直す**: 勢いで進めない。節目ごとに一歩引いて全体を俯瞰する。
- **迷ったら確認する**: 指示が曖昧なとき、方針に迷うときはユーザーに確認を取る。推測で進めない。

## 進め方

タスクの規模に応じてフローを使い分ける。

**単純な修正**（typo、1箇所の変更など）: そのまま着手してよい。

**中規模以上のタスク**（3ステップ以上・複数ファイル変更）:
1. **プランを立てる**: プランモードに入り、方針を決める。いきなりコードを書かない。
2. **todoをファイルに書き出す**: 全体像・作業順序・依存関係をファイルに保存する。冒頭に方針・判断根拠を簡潔に記載し、設計メモとしても機能させる。保存先はプロジェクトの規約に従う（規約がなければ `.claude/todo/`）。
3. **todoを書く段階で以下に気づいたら → 設計書を書く**:
   - 方針の選択肢が複数あり、トレードオフの検討が必要
   - 新規モジュール/パッケージの追加が必要
   - 既存インターフェース（API・DBスキーマ等）の変更が必要
   - 設計書には何を作るか（仕様）・どう作るか（設計）をまとめ、判断根拠を明文化する。保存先はプロジェクトの規約に従う（規約がなければ `.claude/design/`）。
4. **進行中の作業を TaskCreate / TaskUpdate で追跡する**: todoファイルが全作業の全体像を管理し、Taskツールは現在着手している範囲に絞って登録・更新する。
5. **todoを見ながら実装する**: 完了したtodoにチェックを付けて進捗を可視化する。方針メモ・設計書と照らし合わせながら進める。乖離があれば立ち返る。
6. **問題が起きたらリプランする**: 無理に突き進まない。立ち止まってプランモードに戻り、todoを更新してから方針を見直す。

## 完了前の検証

タスクを完了とする前に、必ず動作確認を行う。「たぶん大丈夫」で終わらせない。

- **差分確認**: main/master ブランチとの diff を確認し、意図しない変更が含まれていないことを確かめる。
- **テスト実行**: 関連するテストを実行し、全てパスすることを確認する。テストがなければ、変更箇所を実際に動かして期待どおりの入出力・挙動になることを確認する（コマンド実行、API呼び出し、ビルド成功など）。
- **ログ・出力確認**: ビルドログ、ランタイムログ、コンソール出力を確認し、警告やエラーがないことを確かめる。
- **方針との突き合わせ**: 実装がtodoファイルの設計メモ・設計書と合致しているか確認する。乖離があれば立ち返る。
- **自己レビュー**: 変更全体を見直し、不要なデバッグコードや TODO が残っていないことを確認する。

## サブエージェント活用

メインコンテキストを清潔に保つため、サブエージェントを積極的に使う。

- **原則**: 調査・探索・分析はサブエージェントに委譲する。メインコンテキストに不要な情報を流し込まない。
- **1エージェント1タスク**: サブエージェントには単一の明確なタスクだけを渡す。複数の目的を混ぜない。
- **独立タスクは並列実行**: 依存関係のないタスクは複数のサブエージェントを同時に起動する。
- **結果は要約して受け取る**: サブエージェントの出力をそのまま展開せず、メインに必要な情報だけを簡潔にまとめる。

## MCP 利用方針

詳細な使い分けは `.claude/mcp-guidelines.md` を参照。以下は常に意識する原則のみ。

- **標準ツール優先**: 検索は WebSearch、フェッチは WebFetch を第一選択。不足時にMCPへ切り替える
- **Serena**: ファイル全体を読まずシンボル単位で読む。編集は `replace_symbol_body` 優先。変更時は `find_referencing_symbols` で影響範囲を確認する
- **ドキュメント参照**: DeepWiki（repo把握）→ Context7（外部仕様）の順。外部ライブラリの実装・設定には Context7 を原則使う
- **Playwright**: WebFetch で不完全なとき（SPA等）に使う
- **MarkItDown**: PDF大/Word/Excel/PPT → MarkItDown。PDF小/画像の視覚理解 → Read
